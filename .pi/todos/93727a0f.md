## Goal
Bring Android scheduled exports closer to iOS reliability and configuration.

## Current gap
Android uses WorkManager and can schedule by minutes/hours/days/weeks, but `ExportWorker` exports only yesterday. It lacks iOS-style lookback days, exact pending-date recovery, and tap-to-retry notifications.

## Key files
- `app/src/main/java/com/healthmd/presentation/schedule/ScheduleScreen.kt`
- `app/src/main/java/com/healthmd/presentation/schedule/ScheduleViewModel.kt`
- `app/src/main/java/com/healthmd/data/scheduler/ExportScheduler.kt`
- `app/src/main/java/com/healthmd/data/scheduler/ExportWorker.kt`
- iOS reference: `ScheduleSettingsView.swift`, `ExportSchedule.swift`, `ScheduledExportCoordinator.swift`, pending export request models

## Acceptance criteria
- Add configurable lookback days for scheduled exports.
- Worker exports all selected past complete days ending yesterday, not only yesterday.
- Persist pending failed date ranges when recoverable conditions occur.
- Notification tap can retry pending dates or route to a retry UI.
- History records target label, file count, partial warnings, and failed dates for scheduled runs.
- Schedule gating respects paywall/unlock state.

Implemented scheduled export parity improvements:
- Added `scheduleLookbackDays` and pending retry dates to settings.
- Worker exports all configured past complete days ending yesterday plus persisted pending failed dates.
- Recoverable failures persist pending dates for future scheduled retries.
- Notifications route users back to Schedule/History for blocked or retryable runs.
- Scheduled history entries include target label, file count, warnings, and failed date details.
- Schedule enablement is gated behind unlock.

Validation: `./gradlew testDebugUnitTest` passes.
