## Goal
Make the Android metric picker/catalog fully explainable against the iOS 171-metric inventory: every iOS metric should be either supported on Android, mapped to an Android equivalent, or listed as unavailable with a reason.

## Current gap
- Android supports ~98 metrics plus a set of unavailable metrics.
- iOS has ~171 metrics across Sleep, Activity, Heart, Respiratory, Vitals, Body Measurements, Mobility, Cycling, Nutrition, Vitamins, Minerals, Hearing, Mindfulness, Reproductive Health, Symptoms, Medications, Other, and Workouts.
- Some iOS categories are absent or collapsed on Android: Medications, Other, per-symptom metrics, dedicated Cycling category, Hearing, and some Mindfulness/State of Mind fields.

## Key files
- Android metrics: `app/src/main/java/com/healthmd/domain/model/MetricSelection.kt`
- Android labels: `app/src/main/java/com/healthmd/presentation/i18n/LocalizedMetrics.kt`
- Android fetcher: `app/src/main/java/com/healthmd/data/health/HealthConnectManager.kt`
- Android docs: `docs/export-contract/health-connect-phase2-mapping.md`
- Android docs: `docs/export-contract/android-phase3-apple-exclusive.md`
- iOS metric source: `/Users/codybontecou/projects/health-md/app/HealthMd/Shared/Models/HealthMetrics.swift`

## Acceptance criteria
- Add/refresh a generated or manually maintained Android↔iOS metric parity table.
- For every iOS metric id, record Android status: supported, mapped/alias, Android-only, Health Connect unavailable, or Apple-exclusive.
- Update `HealthMetrics.unavailableMetrics` so user-facing unavailable examples cover all missing iOS categories, not just the original export-contract gaps.
- Add dedicated unavailable entries for major absent groups: Medications, Other HealthKit signals, per-symptom metrics, Hearing, State of Mind, and missing mobility/running dynamics.
- Add supported Android metric entries where Health Connect-backed data already exists but is not individually selectable (e.g. `mindful_sessions` if fetched, cycling power/speed/cadence as distinct metric semantics if appropriate).
- Add tests that compare Android supported+unavailable metric ids against the iOS parity table so future changes do not silently regress.
- Update docs with explicit product stance for metrics that are unavailable because Health Connect lacks the record.

## Notes
This ticket is a catalog/UX parity task. Only implement new Health Connect reads when they are low-risk and already supported by the current dependency; otherwise create follow-up implementation todos.
