## Goal
Expose the custom Markdown template editing workflow that Android already partially models/exports, matching iOS user-facing controls.

## Current gap
- Android `MarkdownTemplateConfig` has `CUSTOM` and `customTemplate`, and `MarkdownExporter` renders custom templates.
- Android `FormatCustomizationScreen` lets users pick the `CUSTOM` style but does not expose a text editor or template preview.
- iOS shows a `TextEditor`, placeholder guidance, and preview when custom style is selected.

## Key files
- Android settings model: `app/src/main/java/com/healthmd/domain/model/MarkdownTemplateConfig.kt`
- Android UI: `app/src/main/java/com/healthmd/presentation/settings/FormatCustomizationScreen.kt`
- Android exporter: `app/src/main/java/com/healthmd/data/export/MarkdownExporter.kt`
- Android strings: `app/src/main/res/values/strings.xml` and localized values folders
- iOS reference: `/Users/codybontecou/projects/health-md/app/HealthMd/iOS/Views/FormatCustomizationView.swift`

## Acceptance criteria
- When `MarkdownTemplateStyle.CUSTOM` is selected, show an editable multi-line custom template field.
- Show placeholder help for `{{date}}`, `{{metrics}}`, `{{sleep_metrics}}`, `{{activity_metrics}}`, `{{#section}}...{{/section}}`, etc.
- Add a compact live preview using representative fixture text or a generated sample.
- Persist edits through `SettingsRepository` and verify exports use the edited template.
- Add reset-to-default affordance for the custom template.
- Add tests for settings persistence and exporter rendering with a custom template.
- Add/refresh localized strings for new UI copy.

## Notes
This is UI/settings parity only; exporter support already exists but should be covered by tests.
