## Goal
Match iOS output path controls: Health subfolder, folder-structure templates, filename templates, and live path preview.

## Current gap
Android has `subfolder`, `folderStructure`, and `folderOrganization` fields, but UI does not expose the full path editor/preview and `ExportRepositoryImpl` ignores `settings.subfolder` when writing aggregate exports.

## Key files
- `app/src/main/java/com/healthmd/domain/model/ExportSettings.kt`
- `app/src/main/java/com/healthmd/presentation/export/ExportScreen.kt`
- `app/src/main/java/com/healthmd/presentation/settings/SettingsScreen.kt`
- `app/src/main/java/com/healthmd/data/storage/ExportRepositoryImpl.kt`
- `app/src/main/java/com/healthmd/data/storage/FileExportManager.kt`
- iOS reference: `ExportTabView.outputSection`, `FilenameFormatEditor`, `FolderStructureEditor`, `SubfolderEditor`

## Acceptance criteria
- UI lets users edit Health subfolder, folder organization template, and filename template.
- Supported placeholders match iOS: `{date}`, `{year}`, `{month}`, `{day}`, `{weekday}`, `{monthName}`, `{quarter}`.
- Aggregate exports write under subfolder + folderStructure.
- Obsidian Bases collision naming works with multi-format export.
- Live path preview updates for selected date, folder structure, format(s), and destination.

Implemented file organization/path preview parity:
- Aggregate exports now write under `subfolder + folderStructure/folderOrganization`.
- Settings UI exposes Health subfolder, folder organization presets, custom folder template, and live path preview.
- Export screen shows selected-date path previews for all selected formats.
- Added `ExportSettingsPathTest` for placeholder/path composition and Obsidian Bases collision naming.

Validation: `./gradlew testDebugUnitTest` passes.
