## Goal
Decide whether Android needs an equivalent to iOS iPhone → Mac Destination, then implement or document as intentionally out of scope.

## Current gap
iOS can send iPhone-configured export jobs to Health.md on Mac over local network. Android currently only writes to local SAF folders.

## Options
1. Mark N/A for Android parity and document rationale.
2. Support direct export to synced folders/providers via SAF only.
3. Implement a cross-platform desktop bridge later (local network, QR pairing, or cloud relay).

## Key references
- iOS: `HealthMd/iOS/Views/SyncSettingsView.swift`
- iOS: `HealthMd/Shared/Sync/SyncService.swift`, `SyncPayload.swift`
- macOS destination: `HealthMd/macOS/HealthMdApp+macOS.swift`, `MacExportJobExecutor.swift`

## Acceptance criteria
- Product decision recorded.
- If implementing: Android UI has a destination selector and readiness/status states.
- If not implementing: docs and parity matrix mark Mac destination as platform-specific N/A, not an unknown gap.
