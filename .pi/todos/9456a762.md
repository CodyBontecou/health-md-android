## Goal
Match iOS export history UX and retry behavior.

## Current gap
Android history is a read-only list. iOS supports recent history details, clearing, failed-date retry, richer source/target/file-count metadata, and partial warning summaries.

## Key files
- `app/src/main/java/com/healthmd/presentation/history/HistoryScreen.kt`
- `app/src/main/java/com/healthmd/presentation/history/HistoryViewModel.kt`
- `app/src/main/java/com/healthmd/domain/model/ExportHistory.kt`
- `app/src/main/java/com/healthmd/data/history/*`
- iOS reference: `ScheduleSettingsView.swift` history section, `ExportHistory.swift`

## Acceptance criteria
- Tap a history item to see details: source, target, date range, counts, file count, failure reason, failed date details.
- Retry failed dates or entire date range using current settings/folder.
- Clear history action with confirmation.
- Record manual, scheduled, shortcut/intent, and future Mac/remote sources distinctly.
- Preserve partial failures/warnings from metric fetches and side-effect writes.

Implemented export history detail/clear/retry:
- History cards open a detail dialog with source, target, date range, counts, file count, failure reason, warnings, and failed dates.
- Added retry for failed dates or the original range using current settings/folder.
- Added clear-history confirmation.
- Added richer history metadata and Room migration.
- Added distinct retry/shortcut/remote source enum values for future flows.

Validation: `./gradlew testDebugUnitTest` passes.
