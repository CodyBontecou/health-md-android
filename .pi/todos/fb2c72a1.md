## Goal
Implement high-value Health Connect fields that iOS exports but Android currently lacks.

## Current gap
Android lacks many non-Apple-exclusive fields present in iOS exports.

## Candidate metrics
- Activity: stand time/hours, swimming distance/strokes, wheelchair distance, move minutes, physical effort, downhill/snow sports if available.
- Heart: walking heart rate average if available; document recovery/AFib as N/A if no equivalent.
- Vitals/respiratory: forced vital capacity, FEV1, peak flow, inhaler usage if available.
- Body: waist circumference.
- Mobility/running: step length, double support, asymmetry, stair speeds, 6-minute walk, walking steadiness, running speed/stride/contact/vertical oscillation/power.

## Key files
- `HealthConnectManager.kt`
- `HealthData.kt`
- `HealthDataFields.kt`
- `MetricSelection.kt`
- exporters
- gap matrix: `docs/export-contract/android-ios-gap-matrix.md`

## Acceptance criteria
- Each added metric is fetched from Health Connect and exported in all relevant formats.
- Frontmatter keys and JSON keys follow iOS contract where semantically equivalent.
- Android-only aliases are preserved where already shipped.
- Unsupported Apple-exclusive metrics are explicitly documented as N/A.

Implemented Phase 2 extended metric support:
- Added Health Connect-backed swimming distance/strokes, wheelchair distance, downhill snow-sports distance, walking HR average, running speed/power.
- Added NutritionRecord fat details, vitamins, and minerals end-to-end through metric picker, filtering, HealthData, HealthDataFields, JSON/CSV/Markdown/Bases.
- Documented unsupported Health Connect beta02 gaps in `docs/export-contract/health-connect-phase2-mapping.md`.

Validation: `./gradlew testDebugUnitTest` passes.
