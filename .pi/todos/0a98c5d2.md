## Goal
Add an Android equivalent of iOS in-app release notes so users see concise post-update changes inside the app.

## Current gap
- iOS uses `HealthMd/iOS/ReleaseNotes.swift` with Notelet to show What’s New after updates.
- Android has Play release notes but no obvious in-app release-notes sheet/dialog.

## Key files
- Android app startup/navigation: `app/src/main/java/com/healthmd/presentation/MainActivity.kt`, `presentation/navigation/HealthMdNavigation.kt`
- Android settings/data persistence: `app/src/main/java/com/healthmd/data/settings/SettingsRepositoryImpl.kt`, `domain/repository/SettingsRepository.kt`
- Android resources: `app/src/main/res/values/strings.xml`
- iOS reference: `/Users/codybontecou/projects/health-md/app/HealthMd/iOS/ReleaseNotes.swift`

## Acceptance criteria
- Persist last-presented version/build in settings/DataStore.
- On app open after an update, show a Material 3 “What’s New” dialog/sheet once per version.
- Include current Android parity/release highlights and link to Settings/Docs if appropriate.
- Do not show during tests/debug capture if that would destabilize automated flows.
- Add tests for version gating/persistence.

## Notes
Lower priority than export/data parity. Keep copy Android-specific; do not mention Mac destination unless Android implements it later.
