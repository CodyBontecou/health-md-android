## Goal
Close remaining Android workout-detail gaps compared with iOS where Health Connect exposes equivalent data, and explicitly document the unavailable pieces.

## Current gap
Android already exports useful workout details: type, duration, calories, distance, elevation gained, HR min/max/avg, speed/pace, cadence, power, laps, segments, and sample arrays. iOS additionally models fields such as indoor/outdoor, metadata, route points, splits, elevation loss, stride length, ground contact time, and vertical oscillation.

## Key files
- Android model: `app/src/main/java/com/healthmd/domain/model/HealthData.kt`
- Android fetcher: `app/src/main/java/com/healthmd/data/health/HealthConnectManager.kt`
- Android exporters: `app/src/main/java/com/healthmd/data/export/JsonExporter.kt`, `MarkdownExporter.kt`, `CsvExporter.kt`, `IndividualEntryExporter.kt`
- Android docs: `docs/features/workout-details.md`
- iOS model: `/Users/codybontecou/projects/health-md/app/HealthMd/Shared/Models/HealthData.swift`
- iOS fetcher: `/Users/codybontecou/projects/health-md/app/HealthMd/Shared/Protocols/SystemHealthStoreAdapter.swift`

## Acceptance criteria
- Audit current Health Connect `ExerciseSessionRecord` and related records for equivalents to iOS workout fields: indoor/outdoor, metadata, route, splits, elevation loss, stride length, ground contact time, vertical oscillation.
- Add model fields and exporter output for any Health Connect-backed data available in the current dependency.
- If route access requires special permissions or is not available, document it as N/A/deferred rather than silently ignoring it.
- Keep JSON schema additive/backward-compatible.
- Update Markdown and individual entry workout files to include new fields when present.
- Add tests using workout fixtures with laps/segments/samples and any newly supported fields.
- Update workout docs with Android-specific limitations.

## Notes
Do not block on metrics already documented as Health Connect-unavailable in beta02; record them in the parity ledger instead.
