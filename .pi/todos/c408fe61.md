## Goal
Expand Android Individual Entry Tracking from a small hard-coded subset to the iOS-style per-metric/per-category feature.

## Completed
- Added per-metric `MetricTrackingConfig`/`metricConfigs` while preserving legacy `enabledMetrics` migration.
- Migrates legacy `blood_pressure` to `bp_systolic` + `bp_diastolic` and preserves legacy workouts/glucose/weight behavior.
- Rebuilt Individual Tracking settings UI around `HealthMetrics.allMetrics` with search, category bulk toggles, suggested/all/none actions, category counts, filename template editing, category folder toggle, and unavailable State of Mind/symptom copy.
- Added timestamped individual entry generation for workouts, sleep stages, steps, heart rate, HRV, blood pressure, blood glucose, SpO2, respiratory rate, body temperature, mindfulness sessions, and weight fallback.
- Added `MindfulnessSessionEntry` capture from Health Connect so sessions can export as individual entries when available.
- Individual entry paths now include `entriesFolder`, category/custom folder handling, `{date}`/`{time}`/`{metric}`/`{category}` template tokens, and avoid double-prefixing.
- Export side effects now use the already-resolved individual entry path so preview byte counts remain accurate.
- Range exports/previews force granular Health Connect reads when enabled individual tracking needs timestamped samples.
- Added unit coverage for legacy migration, path/category folder generation, suggested metrics, and representative entry content.

## Validation
- `./gradlew testDebugUnitTest --no-daemon` ✅
- `./gradlew lintDebug --no-daemon` still fails on pre-existing localization/plural issues (first failure: `values-ro/strings.xml` missing Romanian `few` quantity for schedule cadence plurals). New individual-tracking strings were added to all locale resource files and no longer appear in lint output.
