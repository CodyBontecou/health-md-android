## Goal
Expand Android Individual Entry Tracking from a small hard-coded subset to the iOS-style per-metric/per-category feature.

## Current gap
- Android UI/export supports only `workouts`, `blood_pressure`, `blood_glucose`, and `weight`.
- iOS supports per-metric configs, category bulk enable/disable, suggested metrics, custom folders, category folders, and filename templates.
- iOS can create timestamped entries for mood/state-of-mind, symptoms, workouts, and vitals; Android should implement what Health Connect supports and explicitly mark true platform/API gaps.

## Acceptance criteria
- Replace or extend `enabledMetrics: Set<String>` with per-metric tracking config compatible with saved-state migration.
- Add category-based UI using `HealthMetrics.allMetrics`, with search, suggested metrics, all/none actions, and category counts.
- Preserve existing four Android tracked metrics during migration.
- Implement entry generation for all Health Connect-backed timestamped data Android already fetches: workouts, blood pressure samples, blood glucose samples, heart-rate samples, HRV samples, SpO2 samples, respiratory samples, body temperature samples, step samples, sleep stage intervals, and mindfulness sessions where available.
- Keep State of Mind and iOS symptom entries unavailable unless Health Connect support is added; show reasons instead of silently omitting them.
- Filename template supports `{date}`, `{time}`, `{metric}`, `{category}` and respects `entriesFolder` + category-folder setting without double-prefixing.
- Export Preview lists individual-entry side effects and byte counts.
- Add tests for settings migration, path generation, category folders, suggested metrics, and representative entry content per supported category.

## Notes
Scope is local file exports only; do not add Mac destination behavior.
