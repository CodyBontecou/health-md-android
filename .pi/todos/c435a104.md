## What changed
- `ExportRepositoryImpl` now invokes `IndividualEntryExporter` when individual tracking is enabled.
- Entry files are written beneath `individualTracking.entriesFolder`, with optional category organization and configured filename template.
- Added sample-aware BP/glucose entries using actual sample timestamps when granular samples are present, with aggregate fallbacks otherwise.
- Workouts and weight entries are also written as supplemental Markdown files.
- Side-effect write failures cause the date export to fail with existing file-write error/history behavior.
- Export Preview lists individual entry side effects with paths, sizes, and content snippets.
