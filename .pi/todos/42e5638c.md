## Goal
Close Android's metric catalog gap with iOS where Health Connect supports comparable data.

## Current gap
Android defines ~61 metrics; iOS defines 171 metrics across 18 categories. Several Android categories are missing entirely or are placeholders without fetch/export support.

## Candidate additions
- Symptoms: Health Connect symptom-like records if available, or mark N/A.
- Vitamins and minerals from `NutritionRecord` micronutrient fields.
- Hearing if Health Connect provides equivalent records.
- Other metrics: UV, falls, insulin, alcohol, toothbrushing/handwashing if available.
- More reproductive fields where Health Connect has data.

## Key files
- `app/src/main/java/com/healthmd/domain/model/MetricSelection.kt`
- `app/src/main/java/com/healthmd/domain/model/HealthData.kt`
- `app/src/main/java/com/healthmd/domain/model/HealthDataFields.kt`
- `app/src/main/java/com/healthmd/data/health/HealthConnectManager.kt`
- exporters under `app/src/main/java/com/healthmd/data/export/`
- iOS reference: `HealthMd/Shared/Models/HealthMetrics.swift`

## Acceptance criteria
- Produce a mapping table: iOS metric → Health Connect record/API → supported/N/A.
- Add supported metrics end-to-end: permission, model, fetch, metric selection, flat fields, JSON/CSV/Markdown/Bases output.
- N/A metrics are documented as Apple-exclusive or absent from Health Connect.
- Metric count and categories in UI reflect additions.

Implemented Phase 2 metric catalog expansion:
- Added supported Health Connect micronutrients and extended activity/heart/mobility metrics end-to-end.
- Added a Health Connect 1.1.0-beta02 mapping table with supported vs N/A iOS metrics.
- UI metric count now reflects the expanded catalog via `HealthMetrics.totalCount`.

Validation: `./gradlew testDebugUnitTest` passes.
