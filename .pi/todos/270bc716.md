## Goal
Provide Android equivalents for iOS App Intents / Shortcuts automation.

## Current gap
iOS exposes 7 App Intents; Android has no manifest actions, broadcast receiver, app shortcuts, or assistant/tasker integration for exporting or summaries.

## iOS actions to map
- Export yesterday
- Export last N days
- Export specific day
- Export date range
- Get health summary for date
- Get last export status
- Set scheduled export enabled

## Candidate Android implementation
- Dynamic/static app shortcuts for common actions.
- Explicit broadcast/service intents for Tasker/automation apps.
- Optional Google Assistant/App Actions if still appropriate.
- Deep links into export/schedule/history screens.

## Key files
- `app/src/main/AndroidManifest.xml`
- new receiver/service classes under `app/src/main/java/com/healthmd/automation/`
- `ExportOrchestrator.kt`
- `ExportHistoryRepository`
- iOS reference: `HealthMd/iOS/AppIntents/*`

## Acceptance criteria
- At minimum, external explicit intents can trigger exports and return/log clear status.
- Export intents use same paywall/quota/folder/settings behavior as manual export.
- Summary/status intents expose structured extras/results where Android allows.
- Security: avoid exporting health data from arbitrary implicit intents without user opt-in.
