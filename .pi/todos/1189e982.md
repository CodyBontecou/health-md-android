## Completed
Android Daily Note Injection now supports the remaining body-section parity surface.

## Implemented
- Added `injectMarkdownSections` to `DailyNoteInjectionSettings` with backwards-compatible decoding.
- Retained Android's historical `createIfMissing = true` default intentionally to avoid migrating existing users from create-on-missing to skip-on-missing.
- Kept legacy `enabledMetrics` for saved-state compatibility, but injector now uses the already-filtered `HealthData` plus `FrontmatterConfiguration`, so Daily Note Injection follows the same Health Metrics selection as aggregate exports.
- Reworked `DailyNoteInjector` to build an injection document and merge it through `MarkdownMerger`.
- Frontmatter-only mode updates YAML while preserving note body.
- Markdown-section mode merges Health.md-managed sections like Health Data, Sleep, Activity, Heart, Vitals, etc. while preserving user preamble/custom sections.
- Missing daily notes can be created with a date title and injected body sections.
- Added a Daily Note Injection UI toggle for metric sections.
- Added unit coverage for no frontmatter, existing frontmatter, user sections, repeated managed-section replacement, create-if-missing false, and custom frontmatter keys.
