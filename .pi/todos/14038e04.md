## Goal
Make Android `WriteMode.UPDATE` preserve user-authored note content like iOS.

## Current gap
`FileExportManager.writeFile()` currently treats `UPDATE` as replace/overwrite with a comment saying full `MarkdownMerger` would be used. iOS merges frontmatter and app-managed sections while preserving custom content.

## Key files
- `app/src/main/java/com/healthmd/data/storage/FileExportManager.kt`
- `app/src/main/java/com/healthmd/data/export/MarkdownMerger.kt`
- `app/src/main/java/com/healthmd/data/storage/ExportRepositoryImpl.kt`
- iOS reference: `HealthMd/Shared/Export/MarkdownMerger.swift`

## Acceptance criteria
- For Markdown exports, update mode merges frontmatter keys and replaces app-managed sections only.
- User sections/preamble remain intact.
- Append/overwrite behavior remains unchanged.
- Non-Markdown formats have explicit documented behavior, likely overwrite or unsupported update.
- Tests cover existing frontmatter, no frontmatter, custom user sections, and app-managed section replacement.

Implemented real Markdown UPDATE mode:
- `FileExportManager` now uses `MarkdownMerger` for existing `.md` files in UPDATE mode.
- `MarkdownMerger` merges frontmatter, replaces app-managed sections, and preserves user preamble/custom sections.
- Non-Markdown UPDATE writes are explicitly treated as atomic regeneration/overwrite.
- Added `MarkdownMergerTest` coverage for frontmatter merge, no-frontmatter notes, custom sections, and section replacement.

Validation: `./gradlew testDebugUnitTest` passes.
