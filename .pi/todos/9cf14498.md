## Goal
Implement Phase 3 of the Android/iOS parity plan by explicitly handling Apple-exclusive / Health Connect-unavailable metrics as N/A instead of leaving them as phantom selectable metrics.

## Scope
- Add supported-vs-unavailable metric catalog metadata for Phase 3 gaps.
- Remove unavailable-only metrics from active selection counts/categories while preserving future category enums.
- Add metric picker UX explaining Android Phase 3 unavailable metrics.
- Document the Phase 3 N/A decision and guard it with tests.

## Acceptance criteria
- P3 Apple-exclusive metrics such as wrist temperature, electrodermal activity, state of mind/mood fields, heart recovery/AFib, and lung/inhaler metrics are listed as unavailable with reasons.
- Unsupported `audio_exposure`/hearing is no longer counted as a selectable supported metric.
- Persisted stale unavailable metric IDs do not inflate enabled counts.
- Metric picker no longer shows empty 0/0 categories and includes an Android Phase 3 notice.
- Unit tests pass.

## Implementation summary
- Added `UnavailableHealthMetricDefinition` and `HealthMetrics.unavailableMetrics` for Phase 3 Apple-exclusive / Health Connect-unavailable metrics.
- Removed unsupported `audio_exposure` from the selectable supported metric catalog and hid empty unavailable-only categories from the metric picker.
- Updated `MetricSelectionState` so stale persisted unsupported IDs do not count as enabled and unsupported toggles are no-ops.
- Added a metric picker Phase 3 notice explaining Android-unavailable iOS fields.
- Documented the Phase 3 N/A decision in `docs/export-contract/android-phase3-apple-exclusive.md` and linked it from the gap matrix / phase 2 mapping.
- Added unit/doc tests for unsupported metric handling and Phase 3 documentation.
